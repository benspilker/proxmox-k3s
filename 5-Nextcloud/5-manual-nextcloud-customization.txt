#### Step 1: Create a New PVC for `config`

You can create a new PVC just for the `config` directory, separate from the `data` PVC.

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: nextcloud-config-pvc
  namespace: nextcloud
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Mi  # This can be a smaller size since it's just for config data
  storageClassName: longhorn




Save this as `nextcloud-config-pvc.yaml` and apply it:


kubectl apply -f nextcloud-config-pvc.yaml




#### Step 2: Update the Temporary Pod to Mount Both PVCs


nano nextcloud-config-temp.yaml

Now, update the temporary pod definition to mount the `nextcloud-config-pvc` at `/var/www/html/config` and the existing `nextcloud-longhorn-pvc` at `/var/www/html/data`. Here’s the updated YAML:


apiVersion: v1
kind: Pod
metadata:
  name: nextcloud-config-temp
  namespace: nextcloud
spec:
  containers:
    - name: nextcloud-config-temp
      image: busybox
      command: ['sleep', 'infinity']  # Keep the pod running
      volumeMounts:
        - name: nextcloud-data
          mountPath: /var/www/html/data
        - name: nextcloud-config
          mountPath: /var/www/html/config
  volumes:
    - name: nextcloud-data
      persistentVolumeClaim:
        claimName: nextcloud-longhorn-pvc
    - name: nextcloud-config
      persistentVolumeClaim:
        claimName: nextcloud-config-pvc


#### Step 3: Apply the Updated Temporary Pod YAML

Now apply the updated pod definition:


kubectl apply -f nextcloud-config-temp.yaml


This should allow the pod to mount the two different PVCs without the conflict that occurs when trying to mount both directories to a single PVC.
